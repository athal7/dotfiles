Foundation

opencode

The runtime that runs everything else. Open source, model-agnostic, scriptable.

Swap models freely sonnet for build, opus for plan, codex for review — and any provider when one is having a bad day
Permission control per-agent tool and bash allow/ask lists — plan agent reads, build agent writes
Open source auditable, forkable, no lock-in
Mobile, no extra app opencode web as a LaunchAgent over Tailscale, installed as a PWA — vs Claude dispatch or OpenClaw chat servers

Skills, agents, and AGENTS.md are becoming standards — the primitives port even if I leave.

Durable record of why and what's next — outside the model's memory. .opencode/context-log.md, gitignored.

Portable across runtimes

Skills follow the agentskills spec — same SKILL.md works in opencode and Claude Code. No fork, no rewrite.

# install any skill from this repo gh skill install athal7/dotfiles commit
Prefer upstream skills

When a CLI has a maintained skill, install it from the source repo instead of writing your own. Updates flow automatically; gotchas track the tool.

# in packages.yaml skills: - repo: BRO3886/ical skill: ical-cli
Right primitive for the trigger

Skill if the trigger is a concrete moment. Instructions if it's continuous. Agent + permission if it's mode-restricted. Asking which primitive fits is the audit move that beats redesigning the skill description.
