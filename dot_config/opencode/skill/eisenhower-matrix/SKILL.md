# Eisenhower Matrix

**Category:** Decision Making
**Source:** Dwight D. Eisenhower (popularized by Stephen Covey)

Use this when you're busy but not making progress on what matters, or when you need to prioritize a backlog of tasks and activities.

## The Four Quadrants

|  | **Urgent** | **Not Urgent** |
|--|-----------|---------------|
| **Important** | **Q1: Do it** — crises, deadlines, pressing problems | **Q2: Schedule it** — deep work, long-term goals, planning |
| **Not Important** | **Q3: Delegate it** — admin, others' requests, some meetings | **Q4: Eliminate it** — busywork, entertainment, avoidance |

### Priority Order
Q2 > Q1 > Q3 > Q4. Important-but-not-urgent tasks (Q2) are where strategic leverage lives.

## Distinguishing Urgent vs. Important

- **Urgent:** Has a deadline or requires immediate reaction (e.g., client response, production bug)
- **Important:** Aligned with long-term goals or high-leverage outcomes (e.g., architecture refactor, learning, strategy)

## Agent Workflow

When asked to apply the Eisenhower Matrix to a list of tasks:

1. **List all tasks** provided.
2. **For each task**, assess: Is it important? Is it urgent? (ask clarifying questions if context is missing)
3. **Place each task** in the correct quadrant.
4. **Recommend the action** for each: Do now / Schedule / Delegate / Eliminate.
5. **Highlight the Q2 items** — these are the highest-value tasks that tend to get neglected.

## Example

| Task | Quadrant | Action |
|------|----------|--------|
| Fix critical production bug | Q1 (Important + Urgent) | Do now |
| Write architecture design doc | Q2 (Important + Not Urgent) | Schedule this week |
| Reply to non-urgent Slack messages | Q3 (Not Important + Urgent) | Delegate or batch |
| Read #random channel | Q4 (Not Important + Not Urgent) | Eliminate |
| Design new feature | Q2 (Important + Not Urgent) | Schedule |
| Attend meeting with no agenda | Q4 (Not Important + Not Urgent) | Decline/Eliminate |

## Key Insight

Most people over-invest in Q1 (fire-fighting) and Q3 (feeling busy), while neglecting Q2 (where real progress happens). Consistently protecting Q2 time is how leverage is built.
