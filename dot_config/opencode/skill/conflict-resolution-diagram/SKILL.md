# Conflict Resolution Diagram

**Category:** Problem Solving
**Also called:** Evaporating Cloud
**Source:** Eliyahu Goldratt, Theory of Constraints

Use this when two parties are stuck in opposing positions and neither side's proposal is fully acceptable. Works for interpersonal conflicts, strategic disagreements, technical debates, and internal dilemmas.

## The Structure

```
[Shared Objective]
     /         \
[Need A]     [Need B]
     |             |
[Demand A]  [Demand B]  ← these are mutually exclusive (the conflict)
```

The key insight: the conflict exists at the **demand** level, but both parties usually share a **common objective**. Finding that shared objective is what makes resolution possible.

## How to Apply (Work Right to Left)

1. **Identify the demands:** What is each side proposing? What are the mutually exclusive positions?
2. **Identify the underlying needs:** What requirement or need does each demand satisfy? Ask "What does this position protect or achieve?"
3. **Find the shared objective:** What goal would be achieved if *both* needs were met? This is usually a higher-level business, team, or personal objective.
4. **Surface assumptions:** What assumptions make each demand feel like the *only* way to satisfy the need? Challenging these assumptions is where solutions emerge.
5. **Build a win-win solution:** Design an approach that satisfies both needs and advances the shared objective.

## Agent Workflow

When two parties are stuck:

1. Ask each side to state their position (demand).
2. Help articulate the underlying need behind each demand — don't accept "because it's right" as a need.
3. Work together to name the shared objective.
4. Surface the assumption that makes each demand feel mandatory.
5. Generate 2–3 alternative approaches that satisfy both needs without requiring either party to capitulate.

## Example

**Conflict:** Should we redesign the website? (Yes vs. No)

```
[Shared Objective: Run a profitable business]
         /                        \
[Need A: Better conversion]   [Need B: Minimize resource risk]
         |                        |
[Demand A: Full redesign]    [Demand B: No redesign]
```

**Assumptions to surface:**
- Demand A assumes: "Only a full redesign will move conversion."
- Demand B assumes: "Any website project will consume excessive resources."

**Win-win solution:** Run small, targeted landing page experiments focused on conversion. Low resource commitment, measurable impact, no full redesign required.

## Tips

- The shared objective is not "compromise" — it's genuine alignment on what both sides actually want.
- If you can't find a shared objective, the conflict may be deeper than a single decision. Escalate.
- Works for internal conflicts too (e.g., "should I take this job or stay?").
