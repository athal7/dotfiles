# Productive Thinking Model

**Category:** Problem Solving
**Source:** Tim Hurson, *Think Better*

Use this when you have a defined problem and want to find creative, well-evaluated solutions — not just the first obvious answer. It combines structured inquiry, divergent thinking, and explicit success criteria.

## The Six Steps

### 1. "What's going on?"
Understand the problem deeply before solving it.

- What exactly is the problem?
- What's the impact if it's not solved?
- What do we already know? What information is available?
- Who is affected?
- What does the future look like when this is solved? *(the "Target Future")*

### 2. "What's success?"
Define success criteria using the **DRIVE** framework:

| Letter | Question |
|--------|----------|
| **D** — Do | What must the solution *do*? |
| **R** — Restrictions | What must it *not* do? (Use Inversion here) |
| **I** — Invest | What resources can be committed? |
| **V** — Values | What qualities should the solution embody? |
| **E** — Essential outcomes | What must be true for the solution to count as a success? |

### 3. "What's the question?"
Generate "catalytic questions" — questions that, if answered, would lead to achieving the Target Future.

Use: *"How might we...?"* or *"How can I...?"*

Focus on the 2–3 most critical questions.

### 4. Generate answers
Brainstorm answers to the catalytic questions. Don't filter yet. Quantity is the goal. Generate at least 10–15 candidate solutions.

### 5. Forge the solution
Evaluate candidates against the DRIVE success criteria from Step 2. Use a Decision Matrix if there are many options. Select the most promising solution, then ask: how can we make it even better?

### 6. Align resources
Build an action plan:
- What specific actions are required?
- Who is responsible for each part?
- What resources are needed?
- What's the timeline?

## Agent Workflow

When helping solve a complex problem creatively:

1. Work through Steps 1–3 with the user to ensure the problem is well-understood and success is defined.
2. Facilitate Step 4 — generate a full list of candidate solutions without judgment.
3. Apply DRIVE criteria to evaluate (Step 5).
4. Build the action plan (Step 6) with specific owners and timelines.

## Example (abbreviated)

**Problem:** Win a difficult but high-value prospective customer.

**Step 1:** Customer has complex requirements; losing them would significantly impact revenue targets. Sales team has been in contact; ease of use is their key concern.

**Target Future:** Customer is onboarded, using the product daily, contract signed.

**Step 2 (DRIVE):**
- Do: Demonstrate product value and support capability
- Restrictions: Cannot promise features not yet built
- Invest: 2 salespeople for 2 weeks + 1 engineer for 1 week
- Values: Transparent, thoughtful, convincing
- Essential outcome: Customer signs a contract

**Step 3 (Catalytic questions):**
- How might we show the product meets their requirements?
- How might we demonstrate ease of use in a memorable way?

**Step 4 (Ideas):** Slide deck, interactive demo, hands-on trial day, reference call with similar customer...

**Step 5 (Winner):** Hands-on trial day with end users + key stakeholders observing → scores highest on DRIVE criteria.

**Step 6 (Action):** Two salespeople + one engineer travel to customer site. Sales VP owns coordination. Engineering manager preps the demo environment by [date].

## Related Tools

- **Inversion** — use in Step 2 (Restrictions) to identify what the solution must not do
- **Decision Matrix** — use in Step 5 to score candidates against DRIVE criteria
- **Abstraction Laddering** — use before Step 1 if the problem statement itself needs reframing
