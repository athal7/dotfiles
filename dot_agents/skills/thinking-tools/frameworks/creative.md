# Creative Problem Solving

Frameworks for generating novel solutions when obvious approaches are exhausted.

---

## Productive Thinking Model

**Use when:** You have a defined problem and want to find creative, well-evaluated solutions — not just the first obvious answer.

**Six steps:**

**1. "What's going on?"**
Understand the problem deeply. What exactly is it? What's the impact if unsolved? What does success look like (the "Target Future")?

**2. "What's success?" (DRIVE)**
| Letter | Question |
|--------|----------|
| D — Do | What must the solution do? |
| R — Restrictions | What must it *not* do? (use Inversion here) |
| I — Invest | What resources can be committed? |
| V — Values | What qualities should it embody? |
| E — Essential outcomes | What must be true for it to count as success? |

**3. "What's the question?"**
Generate "catalytic questions" — "How might we...?" questions that, if answered, would achieve the Target Future. Focus on the 2–3 most critical.

**4. Generate answers**
Brainstorm without filtering. Quantity is the goal — at least 10–15 candidates.

**5. Forge the solution**
Evaluate candidates against DRIVE criteria. Use Decision Matrix for many options. Ask: how can we make the winner even better?

**6. Align resources**
Build an action plan: specific actions, owners, resources, timeline.

**Related:** Inversion (use in Step 2 Restrictions), Decision Matrix (use in Step 5 to score candidates), Abstraction Laddering (use before Step 1 if problem statement needs reframing)

## Zwicky Box (Morphological Box)

**Use when:** Standard brainstorming is exhausted; the problem has multiple independent dimensions; you need to explore a large solution space systematically.

**Core:** Decompose the problem into independent attributes, list possible values for each, then force combinations to discover non-obvious solutions.

**Steps:**
1. Define the problem clearly.
2. Identify 4–7 independent attributes (dimensions of the solution that can vary independently).
3. For each attribute, list 3–6 possible values — include unconventional options.
4. Generate combinations: pick one value per attribute. Do this randomly (to force unexpected combinations), systematically (to cover the space), or focused (fix 1–2 interesting values and vary the rest).
5. Evaluate each combination as a concrete solution concept.
6. Flag the 1–2 most promising for further development.

**Example — improving customer feedback collection:**

| Collection | Timing | Format | Incentive | Analysis |
|------------|--------|--------|-----------|----------|
| Email survey | On cancellation | Open text | Gift card | AI sentiment |
| In-app popup | After key action | Multiple choice | Discount | Manual review |
| Phone interview | Weekly | Audio | Early access | Trend analysis |
| Social listening | During usage | Video | Public recognition | Team discussion |

Combination: In-app popup + after key action + multiple choice + discount + AI sentiment → triggered micro-surveys with automated tracking.

**Tips:** Volume of combinations matters — most won't be good but some will be unexpectedly strong. Don't dismiss strange combinations immediately.

**Related:** Abstraction Laddering (make sure you're solving the right problem first), Productive Thinking Model (fuller structured process), Decision Matrix (evaluate promising combinations)

## Abstraction Laddering

*(Also in problem.md — listed here because it's often the entry point for creative reframing)*

**Use when:** Stuck on an approach and need to find a different solution space.

Ask "Why?" to move up to more abstract framings. Ask "How?" to descend toward concrete solutions. Moving one level up often opens entirely new solution spaces.

**Quick reminder:**
```
↑ "Why?" → broader scope, different framing
```
